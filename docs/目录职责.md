典型的 **FastAPI 后端分层项目**。可以按这条主线理解：

请求进来 → `api` 接口层 → `schemas` 校验数据 → `core` 提供数据库/认证能力 → `models` 操作表 → `services` 做业务逻辑。

**核心目录**

`app/`：项目主代码目录。

[app/main.py](<C:/Users/86198/Desktop/寻找擅长领域/ai-knowledge-base/app/main.py:18>)：应用入口，创建 `FastAPI` 实例，启动时初始化数据库，注册异常处理器，并挂载各个接口路由。

`app/api/`：接口层，也就是 URL 路由所在目录。

- `auth.py`：注册、登录接口。
- `knowledge_bases.py`：创建、查询知识库。
- `documents.py`：上传文档、解析文档、切块、向量化入库。
- `chat.py`：知识库问答、聊天历史。

`app/schemas/`：数据结构层，用 Pydantic 定义请求体和响应体。

- `auth.py`：注册、登录、Token、用户返回结构。
- `knowledge_base.py`：知识库创建和返回结构。
- `document.py`：文档返回结构。
- `chat.py`：提问、回答、引用片段、历史记录结构。

`app/models/`：数据库模型层。

[app/models/entities.py](<C:/Users/86198/Desktop/寻找擅长领域/ai-knowledge-base/app/models/entities.py:10>) 里定义了数据库表：

- `User`：用户表
- `KnowledgeBase`：知识库表
- `Document`：文档表
- `DocumentChunk`：文档切片表，里面有 `pgvector` 向量字段
- `ChatRecord`：问答历史表

`app/core/`：基础设施层，放全项目通用能力。

- `config.py`：读取环境变量，比如数据库地址、JWT 配置、Ollama 模型名。
- `database.py`：数据库连接、Session、初始化表。
- `security.py`：密码哈希、JWT 创建和解析。
- `dependencies.py`：FastAPI 依赖，比如获取当前登录用户、检查知识库归属。

`app/services/`：业务服务层，放不直接属于接口和数据库表的核心逻辑。

- `file_parser.py`：解析 TXT、MD、PDF 文件。
- `text_splitter.py`：把文档内容切成 chunk。
- `ollama_client.py`：调用 Ollama 做 embedding 和 chat。
- `rag.py`：组装 RAG 提示词。

**辅助目录**

`tests/`：自动化测试目录。测试接口、安全逻辑、文件解析、文本切分等。

`docs/`：项目文档目录。包括接口说明、数据库设计、演示说明。

`samples/`：示例材料目录。比如 `demo_knowledge.md` 可以用来测试上传知识库文档。

`.venv/`：本地 Python 虚拟环境，不属于项目业务代码。

`.pytest_cache/`、`__pycache__/`：Python/pytest 运行产生的缓存，可以忽略。

**根目录文件**

`Dockerfile`：定义 API 服务镜像怎么构建。

`docker-compose.yml`：一键启动 API、PostgreSQL/pgvector、Ollama。

`requirements.txt`：Python 依赖包。

`.env` / `.env.example`：环境变量配置。

`README.md`：项目说明文档。

一句话总结：  
这个项目的核心就是 `app/api` 负责接请求，`app/services` 负责 RAG 业务处理，`app/models` 负责数据库表，`app/core` 负责配置、数据库和认证这些底层能力。
